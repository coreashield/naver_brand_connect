# 블로그 원고 자동 작성 프로그램 프로젝트 계획서

## 1. 프로젝트 개요

### 1.1 목적
네이버 브랜드커넥트 제품 링크를 Playwright MCP를 통해 분석하고, SEO 최적화된 블로그 원고를 자동 생성하는 프로그램 개발

### 1.2 실행 환경
- IDE: Visual Studio Code
- 실행 도구: Claude Code
- 웹 스크래핑: Playwright MCP

---

## 2. 핵심 기능 요구사항

### 2.1 제품 분석 (Playwright MCP)
| 항목 | 설명 |
|------|------|
| 입력 | 네이버 브랜드커넥트 제품 URL |
| 수집 정보 | 제품명, 가격, 상세 설명, 이미지, 스펙, 리뷰 요약 |
| 출력 | 구조화된 제품 데이터 (JSON) |

### 2.2 원고 생성
| 항목 | 설명 |
|------|------|
| 형식 | 제목 / 소분류(소제목) 구조 |
| 글자 수 | 1,500 ~ 2,000자 |
| 스타일 | SEO 최적화, 자연스러운 구매 유도 |
| 톤앤매너 | 친근하고 신뢰감 있는 리뷰 형식 |

### 2.3 이미지 묘사 삽입
원고 사이에 텍스트로 이미지 묘사 삽입
- 촬영 각도
- 분위기/조명
- 촬영 위치
- 포즈/구도
- 소품/배경

**예시:**
```
(블로그 주인이 해당 제품을 손에 들고 창가 자연광 아래에서 45도 각도로 촬영한 클로즈업 컷, 배경은 화이트톤 인테리어)
```

| 구분 | 기술 |
|------|------|
| 런타임 | Node.js |
| 웹 스크래핑 | Playwright MCP |

## 5. 프로젝트 구조

```
blog-writer-project/
├── src/
│   ├── scraper/
│   │   └── productScraper.js    # Playwright MCP 제품 분석
│   ├── generator/
│   │   └── contentGenerator.js  # 원고 생성 로직
│   ├── templates/
│   │   └── blogTemplate.js      # 원고 템플릿
│   └── index.js                 # 메인 실행 파일
├── output/                      # 생성된 원고 저장
├── config/
│   └── config.js                # 설정 파일
├── package.json
└── README.md
```

---

## 6. 실행 흐름

```
1. 사용자가 제품 URL 입력
          ↓
2. Playwright MCP로 페이지 접속 및 데이터 수집
          ↓
3. 수집된 데이터 구조화 (JSON)
          ↓
4. Claude를 통해 원고 생성
   - SEO 키워드 추출
   - 제목/소제목 구성
   - 본문 작성 (1,500~2,000자)
   - 이미지 묘사 삽입
          ↓
5. Markdown 파일로 출력
```

## 8. 사용 방법 (예상)

```bash

cd blog-writer-project
node src/index.js --url "https://brandconnect.naver.com/..."

```


## 9. 출력 예시

### 입력
```
URL: https://brandconnect.naver.com/904249244338784/affiliate/products/899248351112384
```

### 출력 (output/원고_20241230_제품명.md)
```markdown
# 겨울 필수템! OOO 제품 솔직 리뷰 (실사용 후기)

(블로그 주인이 택배 박스를 개봉하며 설레는 표정으로 카메라를 바라보는 모습, 
거실 소파에 앉아 자연광이 들어오는 오후 시간대, 전신이 보이는 중간 거리 샷)

## 이 제품을 선택한 이유

요즘 OOO 때문에 고민이 많으셨죠? 저도 마찬가지였어요...
[본문 계속]

(제품을 손에 들고 디테일을 보여주는 클로즈업 샷, 
흰색 대리석 테이블 위에 제품을 올려놓고 45도 위에서 촬영, 
부드러운 간접 조명으로 제품의 질감이 잘 드러나는 구도)

## 실제 사용해보니

[본문 계속 - 총 1,500~2,000자]
```
